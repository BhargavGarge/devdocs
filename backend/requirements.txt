# Direct dependencies only — pip resolves compatible transitive versions itself.
# (Previously a raw `pip freeze` of the whole dev venv, which pinned unrelated
# packages like langchain-core/langchain-groq/ragas left over from earlier
# experiments — those aren't imported anywhere in app/ and conflicted with
# each other on a clean install. See imports in app/**/*.py for what's real.)

# CPU-only PyTorch build. Without this, `pip install torch` on Linux defaults
# to the CUDA-enabled wheel, which pulls in several GB of nvidia-* runtime
# packages (cublas, cudnn, triton, etc.) that are useless without a GPU and
# OOM a 512MB host just loading them — this is what was killing the Render
# free-tier deploy. PEP 440 sorts "+cpu" local versions higher than the
# bare version, so adding this extra index (without pinning torch itself)
# is enough for pip to prefer the CPU wheel.
--extra-index-url https://download.pytorch.org/whl/cpu
